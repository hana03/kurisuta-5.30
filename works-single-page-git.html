Git

Git初心者はまずこれを理解しよう！Gitとは？GitについてとGitの用語を詳しく解説します。

こんにちは。駆け出しエンジニアのRIKAです。私がGit学習を始めた頃、Gitについての解説を聞いても知らない国の言葉を聞いているような気分でした。
学習に入る前に準備が必要かなと感じました。英文を学ぶ前に英単語を学習するように。

私は、まず用語の意味を理解しました。聞きなれない言葉もひとつひとつの意味を理解していけば、解説動画もスムーズに耳に入ってくるようになりました。

今回は、GitについてとGitの用語を初心者向けに解説していきます。

Gitとは？

データ編集時、編集前のデータをコピーして念の為に保存、編集後、再びコピーして保存！なんてことしていませんか？
そんな時に便利なのが、バージョン管理システムのGitです。Gitは変更前の履歴も、変更後の履歴も全て管理してくれるシステムです。

バージョン管理システム

バージョン管理システムとは、いつ、誰が、何のために変更をしたのかを記録してくれるシステムのことです。

Gitではコミットという「セーブポイント」をたくさん設ける事が出来ます。それにより、いつでも以前の状態へ戻すことが可能になります。間違えや変更も何度でもやり直しができます。

Gitで何ができるの？

ファイルを一括で管理できる

Gitでは、コミットを辿ることで変更前の状態に何度でもさかのぼる事が出来ます。

ファイルの変更履歴を複数人で共通できる

コミットメッセージを残す事で、いつ、誰が、何のために何を変更したのかを残す事が出来ます。
一人で作業をしていたとしても、何で変更したんだっけ？何てことありますよね。コミットメッセージを残すことで、一人で作業を進める場合でも、複数人での作業を進める場合でも効率良く作業を進める事ができます。

過去の履歴に簡単に戻れる

データ編集時、昨日のあの変更は余計なことだった！昨日のデータまで戻りたい！何てことありませんか？Gitでコミットを残していれば、昨日の状態に戻せます。

Gitの基本用語を知ろう

リポジトリ

Gitにより管理されている、フォルダの事です。
通常のフォルダとの違いは「.git」フォルダが中に含まれているかどうかで判断出来ます。

コミット

前回の履歴から現在の状態までの変更を記録します。
コミットは時系列順に繋がった状態でリポジトリに格納されます。このコミットを辿ることで、過去の変更履歴を確認することが出来ます。

ステージ

リポジトリへコミットする準備の為の場所です。
変更されたファイルを選択し、ステージへ移動させます。

ブランチ

ブランチとは、枝別れの事です。
保存を分岐する事で、「変更A」「変更B」と分けて作業を進める事が出来ます。ブランチごとに目的を分ける事でそれぞれの作業に影響を与えることなく、別々の作業を進める事が可能になります。

マージ

マージは、枝別れさせたブランチを合体させる手段です。
ブランチを分けて作業を進めた、「変更A」と「変更B」を合体させる事が出来ます。

コンフリクト

ブランチを分けて作業をしていると、同じ場所を変更してしまう事もあります。そのままコミットした場合二つのデータがぶつかり合ってしまい、コンフリクト（マージで競合）が起きます。

GitとGithabの違いは？

GitとGithabは別物なの？と疑問に思う方もいらっしゃるかもしれません。はい！別物です。

Git

変更の履歴を残す事ができる、バージョン管理システムの事です。

Githab

世界中で多く利用されているソフトウェア開発プロジェクトのためのWebサービスです。
